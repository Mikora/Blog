做一个web容器

发表于 2017-03-06

1.浏览器带的功能，定制功能

wk和ui的不同容器，cookie管理，预置信息等

1.wk和ui对比

内存对比：

地址 uiwebview wkwebview
www.baidu.com 100mb 10mb
www.taobao.com 100mb 5mb
www.qq.com 130mb 5mb

加载时间:

地址 uiwebview wkwebview
www.baidu.com 2300ms 1700ms
www.taobao.com 1400ms 1000ms
www.qq.com 2500ms 1300ms

显而意见，wk在各个方面都优于ui，所以我们需要在iOS8及以上的设备支持wk
如果需要支付7或者以下的设备，可以自己定义一个view包含wk和ui，由设备版本去决定加载谁

3.原生的功能组件

基本的容器都应该有一些基本的功能

  1. 设备信息
  2. 图片相关
  3. gps地图相关
  4. 上传下载
  5. app相关，推送，生命周期
  6. 加速计相关
  7. 其他定制

这里我们要考虑的不仅是如何实现这些组件，更重要的是如何让容器更加友善的集成这些组件，以便于扩展或者根据业务需求变更组件功能

4.提升加载速度

1.缓存

缓存可以使用请求的状态304做，但是需要自己去截取请求，缓存下来，下次请求的时候，304直接从缓存中取

还可以直接做成缓存文件，由本地组件统一控制，启动或请求时查看并修改所有不符合当前版本的缓存

2.离线包＋预加载

当一个业务页面太大，我们希望第一次的时候就直接从本地获取，或者第一次加载完从本地获取，就可以用到离线包，内置在工程里面，请求的时候直接定向到本地路径。

当一个页面大量被用到，并且要求加载时候足够短，我们可以预加载到内存池里面，当需要的时候直接从内存中读取。

3.WebView启动和发送请求改成并行

webview是可以直接loadDatade，所以初始化和请求可以同时进行，从而不使用以前的线初始化，再loadrequest的方式。

5.安全问题

1.防止运营商截取

有时候页面会出现奇怪的广告，或者直接定向到其他地方了，解决方法https

2.防止离线包被篡改

有人会把下载离线包的链接定向到自己的链接上，这样他就可以窜改我们包里面的内容，为了防止，我们可以在请求更新规则上加上一个md5或者其他生成规则的安全码，验证下离线包的安全性

3.防止js注入调用工程里面的其他敏感信息

严格控制容器js所能提供的能力，比如调用一个类的方法，确定是原生可以web不可以调起，在调用时有权限管理
